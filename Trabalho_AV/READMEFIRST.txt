METODOLOGIA RAD

1º Fase: Requisitos.

Propósito: fazer sistema de cadastro de produtos de uma Loja de Acessórios.

Nome da loja: Lojinha Moonli
Cores principais da loja: amarelo(#FFE282), rosa(#FE7698), e roxo(#FFE282).

Escopo: Vamos precisar de- SKU(número chave);
                           Nome do Produto;
                           Descrição;
                           Marca;
                           Preço;
                           Preço com +10%;
                           Quantidade disponível;
                           Garantia;
               
Interface do Sistema: Sistema projetado para operar no Windows 10/11, 64 bits, 8GB RAM.

Interface do Usuário: A GUI do sistema possui botões e caixas de texto , facilitando o controle por teclado e mouse.

Interface de Software: O software permite cadastrar utilizando o SQLite, os dados informados.

Requisitos Funcionais: 
     • O usuário deve ser capaz de cadastrar o produto;
     • O usuário deve ser capaz de ler os dados de um produto;
     • O usuário deve ser capaz de atualizar os dados de um produto;
     • O usuário deve ser capaz de excluir um produto.
     • O usuário deve ser capaz de limpar as caixas de texto.

2º Fase: Design de Interface do Sistema.

Criar 3 files, para poder otimizar e organizar o projeto.
file 1: banco, para criar a tabela e a conexão com o banco.
file 2: view, para criar e dar vida as funções dos botões que vão precisar ser criados no frontend.
file 3: frontend, para criar toda nossa parte visual, com os botões funcionando, as entradas de texto, e a visualização dos dados.

- na nossa interface tera o nome da loja no topo com uma mensagem, entradas para os dados a esquerda, e visão dos dados entrados á direita.
